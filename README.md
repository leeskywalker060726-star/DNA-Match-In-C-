## 1. 评分原理研究

根据对 Lab 1 评分系统的多次提交测试与文档研究，比对质量的评分标准主要基于以下三个维度：**匹配总长度**、**分段内编辑距离**以及**分段数量**。

### 1.1 核心评分模型
本次实验采用的核心评分公式可抽象为：
$$TotalScore = \sum_{segments} (Length_i - EditDistance_i)$$

通过实验总结，该模型生效需满足以下硬性约束：
1. **长度阈值**：分段长度 $L \ge 30$ 。
2. **准确率阈值**：编辑距离 $ED$ 必须满足 $ED < 10\% \times L$（即准确率 $Accuracy \ge 90\%$）。
3. **负奖励机制**：分段数量并不影响评分，不过分解过碎会导致段落无效。

### 1.2 变异处理逻辑
由于 DNA 变异（突变、插入、删除、倒位、移位等）具有复杂性 ，算法并不试图还原具体的生物变异步骤，而是通过**最大化得分路径**来隐式覆盖这些变异。例如，对于移位突变，算法会将其识别为两个独立的高分匹配段。

## 2. 算法设计思路

### 2.1 传统算法的局限性
传统的 Smith-Waterman 算法在处理数万级别的序列时，其 $O(mn)$ 的时间复杂度会导致严重的性能瓶颈。在本实验中，如果对每一个 $Q$ 的片段都扫描一遍 $R$，计算量将达到 $10^{10}$ 级别。

### 2.2 优化后的滑动窗口 DP
本算法采用**一维动态规划**结合**增量式滑动窗口**的策略：

1. **状态定义**：$f[i]$ 表示 Query 前 $i$ 个碱基所能获得的最大累积得分。
2. **状态转移**：对于每个起点 $i$，枚举长度 $j \in [30, 120]$，下界30是有效段落长度值，而在本实验中长度>60的段落都没有必要存在，但是实际实验发现改成120可有效提升得分，具体原理还没有探究出来。
3. **增量更新（核心提速）**：
   - 当窗口长度从 $j-1$ 增加到 $j$ 时，无需重新计算编辑距离。
   - 只需检查新进入窗口的碱基 $Q[i+j-1]$ 与 $R$ 对应位置是否匹配。
   - 这使得单次匹配计算从 $O(L)$ 降至 $O(1)$。
4. **反向互补处理**：在滑动窗口过程中，同时维护正向匹配计数 `fw[]` 和反向互补匹配计数 `rv[]` 。
5. **贪心合并**：在回溯阶段，如果相邻的 DP 状态是由完美匹配（$ED=0$）构成的，则进行贪心合并以减少分段数量。

## 3. 核心算法伪代码

```cpp
// 初始化 DP 数组
for i from 0 to lenQ:
    f[i] = 0, g[i] = -1

for i from 0 to lenQ - 1:
    // 允许跳过当前碱基（处理插入/垃圾序列）
    if f[i] > f[i+1]:
        f[i+1] = f[i], g[i+1] = i, type[i+1] = SKIP

    // 初始化滑动窗口计数器
    memset(fwd_counts, 0); memset(rev_counts, 0)

    // 枚举窗口长度 j
    for j from 1 to 120://这个窗口在query2-ref2中可以改为60，时间复杂度减半但得分不变
        char c = Q[i+j-1]
        char rc = getComplement(c)
        
        // 更新 R 中所有可能起始点 k 的匹配计数
        for k from 0 to lenR - j:
            if R[k+j-1] == c: fwd_counts[k]++
            // 反向互补计数逻辑 (基于位置 k 的前向关系)
            rev_counts[k] = rev_counts[k+1] + (R[k] == rc)

            // 达到计分阈值时更新 DP
            if j >= 30:
                edits = min(j - fwd_counts[k], j - rev_counts[k])
                if edits < j * 0.10: // 准确率 >= 90%
                    score = j - edits
                    if f[i] + score > f[i+j]:
                        f[i+j] = f[i] + score
                        g[i+j] = i
                        best_r_pos[i+j] = k
```

## 4. 时空复杂度分析

### 4.1 时间复杂度
- **外层循环**：遍历 Query 序列，复杂度 $O(m)$。
- **中层循环**：窗口长度 $K$（常数 120），复杂度 $O(K)$。
- **内层循环**：更新 Reference 的匹配状态，复杂度 $O(n)$。
- **总复杂度**：$O(m \cdot K \cdot n)$。但在实现中，由于 $K$ 是极小的常数且内层仅为简单的加法运算，实际运行效率远高于传统的 $O(mn)$。
- **实测性能**：对于 Lab1 query1-ref1 规模的数据，运行时间约为 **0.542549s**；对于大规模数据（如Lab1 query2-ref2），受cpu频率变化影响，每次测试速度幅度较大，但是可以保证在（i9-12900HX 32G）290s内跑完，且据实验发现query2-ref2中把窗口大小改为60，运行时间可以减半但得分不变

### 4.2 空间复杂度
- 使用了静态数组存储 `f`, `g`, `fw`, `rv`，总空间开销为 $O(m + n)$。
